프로젝트를 진행하면서 다음의 세 가지 규칙을 지키기로 한다.

1. '진행상황' 폴더 속 두 가지 log파일
  - '소스 코드 log'는 오직 소스코드의 수정, 추가, 삭제 등에 관한 내용을 올리고, 누구든지 작업 후 해당 텍스트 파일에 작업 내역을 추가한다.
  - '활동 log'는 회의 시, 혹은 일상 대화 중 결론내린 작업에 대한 기록을 남긴다.

2. 깃허브 마일스톤 활용하기
 - 누구든지 이슈에 질문, 혹은 요청 사항을 쓰게 되면 이를 추후 모임 시 처리한다. 
 - 추후 모임 시 그 날에 해당하는 마일스톤을 만들고 처리가 요구되는 이슈들을 해당 마일스톤으로 옮겨놓는다. 
 - 당일 모임 시 문제를 해결하며 진행상황을 기록한다. 물론 모임 전 이슈들은 일상의 대화 등과는 구별되어야 할 것이다.

3. 이슈 사용하기
  질문, 요청 혹은 잡담으로 사용할 수 있다.
 
ex) 이슈: 제가 소스코드 조금 수정했습니다. 한 번 확인해주세요.
    답변: 괜찮네요! 로그 기록하시고 그대로 ㄱㄱ 합시다.
    -> 본인 로그를 수정하고 그대로 이어 나간다.

    이슈: 제가 소스코드 조금 수정했습니다. 한 번 확인해주세요.
    답변: 일단 로그 수정하시고, 이건 다들 모여서 상의가 필요할 것 같습니다.
    -> 다음 모임 때 모여서 당일 모임 용 마일스톤에 추가하고, 문제에 대해 상의하고 해결 결과를 기록한다.

    이슈: 안뇽~
    -> 그냥 무시해도 된다.

    이슈: 제가 테스트 용으로 프로그램 만들어서 추가해놨습니다. 한 번 돌려보세요.
    답변: 이거 다음 모임 때 다시 보고 싶어요!
    -> 다음 모임 때 모여서 당일 모임 용 마일스톤에 추가하고, 문제에 대해 상의하고 해결 결과를 기록한다.

    이슈: 제가 테스트 용으로 프로그램 만들어서 추가해놨습니다. 한 번 돌려보세요.
    답변: 궁금하지 않아요.
    -> 그냥 무시해도 된다.
